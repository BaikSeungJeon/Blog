### Sequence

시퀀스는 세 가지가 있습니다. <code>List</code>, <code>Tuple</code>, <code>Range</code>

#### List
> 객체를 순서대로 저장하는 집합형 자료구조

```python
# 리스트 예제

myList = list()         # 요소가 없는 list 객체 생성
myList = [1, 3.14, 100] # literal로 표현
print(myList)           # [1, 3.14, 100]

# 중첩 리스트도 가능
myList = [1, 2, 3, [3.14, 5.25, 100], 5, 6]

# 출력
print(myList)       # [1, 2, 3, [3.14, 5.25, 100], 5, 6]
print(myList[0])    # 1
print(myList[-1])   # 6
                    # 인덱싱에 '-'가 있으면 뒤에서부터 계산
print(myList[3][2]) # 100
```

```python
# 슬라이스
# 슬라이스는 잘라도 원본과 결과본의 데이터 타입이 같다
# 소시지를 반으로 잘라도 소시지인 것처럼

print(myList[0:2])  # [1, 2]
                    # 슬라이싱에 앞은 포함, 뒤는 불포함

print(myList[0:1])  # [1]
                    # 얘는 당연히 1이 나올 것인데,
                    # 리스트이니까 [1]로 출력됨

print(myList[1:])   # [2, 3, [3.14, 5.25, 100], 5, 6]
                    # 1 이후로 전체 출력

print(myList[:2])   # [1, 2]
                    # 위와 반대

print(myList[:])    # [1, 2, 3, [3.14, 5.25, 100], 5, 6]
                    # 복사할 때 사용
```

```python
# 리스트 연산

a = [1, 2, 3]
b = [4, 5, 6]
c = a + b

print(c)  # [1, 2, 3, 4, 5, 6]
          # 파이썬은 벡터 연산이 안 됨
          # 즉 [5, 7, 9]가 나오는 게 아니라
          # 위처럼 리스트가 연결된 값이 출력

c = a * 3
print(c)  # 마찬가지로 [3, 6, 9]가 아니라
          # [1, 2, 3, 1, 2, 3, 1, 2, 3]이 출력
```

```python
# 리스트는 객체
# 즉 많은 메소드를 가지고 있다

a = [1, 2, 3]
a.append(4)
print(a)  # [1, 2, 3, 4]

a.append([4, 5])
print(a)  # [1, 2, 3, 4, [4, 5]]
```
